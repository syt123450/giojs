Development Setup
  • 1. Fork the repo.
  • 2. Clone your own Github repo to your local machine.
    • git clone https://github.com/syt123450/Gio.js.git
  • 3. Install dev tools/libs
    • npm install # yarn support is not here yet
Project Structure
  • 1. assets

    • Static resource and production dependencies (stat.min.js & three.min.js)

  • 2. build

    • Contains files for distribution. This folder will only be updated if there is a new release. Latest development is reflected in branches.

  • 3. examples

    • Carefully selected examples to illustrate major functionalities of the API.

  • 4. src

    • Source code.

  • 5. test

    • Unit tests.

  • 6. website

    • Source code for the official website of Gio.js.

Pull Request Guidelines
  • 1. Refactor/Optimization/Bug fix
    • Add (fix #xxxx[,#xxx]) (#xxxx is the issue id) in your PR title for a better release log,

      • e.g. update entities encoding/decoding (fix #3899).

    • Provide detailed description in the PR.
    • Add relevant tests.
    • Run tests. Make sure newly added code does not break existing functionalities.
      • npm test
  • 2. New feature
    • "Provide convincing reason to add this feature. Ideally you should open a suggestion issue first and have it greenlighted before working on it."
    • Create a new branch
      • git checkout -b my-new-feature git commit -am 'Description for new feature' git push origin my-new-feature
    • Add test cases
    • Don't forget to submit a pull request :P (This only contains simplest, copy from https://markvis.js.org/#/?id=motivation, we may add more to this page)
